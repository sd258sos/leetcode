编译和解释为什么分开
当java虚拟机启动的时候 解释器可以先发挥作用 不必等即时编译器全部编译后再来执行 这样可以省去不必要的执行时间 随着时间的推移 编译器发挥作用 把越来越多的代码编译成本地代码 获得更高的效率
jit 执行的是热点代码 就是那种使用频率比较高的 比如 for循环的
